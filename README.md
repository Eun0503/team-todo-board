# Team 공용 Todo Board

본 프로젝트는 팀 프로젝트나 스터디 그룹원들이 다 함께 목표를 공유하고 진행 상황을 체크하는 **'공용 화이트보드' 컨셉**의 투두 어플리케이션입니다.
팀원 누구나 접속하여 팀의 새로운 할 일을 추가할 수 있고, 누군가 특정 작업을 완료(체크)하면 다른 팀원들도 접속했을 때 해당 작업이 완료되었음을 즉시 확인할 수 있습니다.

특히 이 프로젝트는 **GitHub Actions(Self-Hosted Runner)와 Docker**를 활용한 완벽한 CI/CD 자동 배포 파이프라인이 구축되어 있습니다.

---

## 🚀 배포 및 자동화 아키텍처 (CI/CD)

- **컨테이너화 (Docker)**: 프론트엔드와 백엔드를 모두 Docker 이미지로 빌드하고, `docker-compose`를 사용하여 하나의 네트워크 환경에서 안정적으로 구동되도록 구축했습니다.
- **자동 배포 (GitHub Actions)**: `Self-Hosted Runner` 방식을 채택하여, 팀원이 `main` 브랜치에 코드를 푸시할 때마다 로컬 배포 서버에서 자동으로 최신 코드를 당겨오고 컨테이너를 재시작하여 무중단(?) 업데이트를 수행합니다.
- **외부 접속망**: `ngrok` 터널링 기술을 활용해 외부(팀원들)에서 로컬 배포 서버 환경으로 접속할 수 있도록 네트워크를 개방하였습니다.

---

## 🛠️ 기술 스택

- **Frontend**: Next.js (App Router)
- **Backend**: FastAPI, SQLite
- **DevOps**: Docker, Docker Compose, GitHub Actions, ngrok

---

## 💻 주요 기능

### 1. 실시간 공용 캘린더 및 일별 투두 조회
- 팀의 진행 상황을 한눈에 파악할 수 있는 주간 네비게이터를 제공합니다.
- 각 요일별로 등록된 투두의 총 개수가 실시간 뱃지로 동기화됩니다.

### 2. 인라인 CRUD (팀원 실시간 공유)
- **추가**: 입력창에 텍스트를 입력하여 할 일을 간편하게 등록합니다.
- **수정**: 수정 버튼 클릭 시 인라인 에디팅 모드로 전환됩니다.
- **토글 및 삭제**: 완료 토글(완료/해제) 및 삭제 동작을 비동기 API 연동으로 매끄럽게 처리하여 새로고침 시 모든 팀원에게 변경사항이 즉각 반영됩니다.

### 3. 필터링 및 로딩/에러 처리
- 전체, 진행 중, 완료 필터 탭을 눌러 원하는 상태의 투두만 조회할 수 있습니다.
- 예외 상황 핸들링 및 스켈레톤 UI 로딩 상태를 제공하여 팀원들에게 부드러운 사용성을 제공합니다.
